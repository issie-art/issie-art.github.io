# 产品设计

## 定位

Issie's Blog 是一个个人技术博客，核心定位是**观点驱动的技术写作**——不是教程堆砌，而是通过抽象思考传递对技术、工程和创作的深度理解。

## 目标受众

- 有一定经验的后端/全栈工程师
- 对技术哲学、工程方法论感兴趣的读者
- 寻求深度思考而非速食内容的阅读者

## 核心功能

| 功能 | 说明 |
|------|------|
| 文章发布 | Markdown 写作，Hexo 渲染为静态 HTML |
| 文章目录 | 文章页自动生成 TOC，桌面端侧边栏 + 移动端悬浮按钮 |
| 归档浏览 | 按时间线查看历史文章 |
| 关于页面 | 个人简介与联系方式 |
| SEO | Open Graph、canonical、结构化数据（JSON-LD） |

## 内容策略

### 文章结构（破-立-证-用）

1. **破**：打破读者的错误认知或常见误解
2. **立**：提出核心论点（锋利、可记忆）
3. **证**：论证为什么这么说的深层逻辑
4. **用**：什么时候该用，什么时候不该用

### 内容比例

- 70% 抽象思考 / 30% 具体故事
- 不堆砌案例，故事只用来点睛

### 语言风格

- 中文为主，技术术语保留英文
- 反思性、技术性、诚实
- 不用感叹号，不用夸张修辞
- 每篇文章要有一个反直觉或反常识的核心论点

## 设计原则

- **极简**：无多余装饰，阅读体验优先
- **清晰**：排版宽松、层级明确、目录辅助导航
- **快速**：静态生成，无服务端依赖，首屏加载快
